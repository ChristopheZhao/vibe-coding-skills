## 1. 需求分析
- 用户意图：
  - 设计一个与 `sdd-plan-maintainer` 协同的“分层项目记忆”skill。
  - 记录更完整的研发关键状态，并向 coding agent 提供按需上下文。
- 范围内：
  - 定义分层记忆模型（L1 项目状态、L2 关键节点、L3 节点洞察）。
  - 定义触发规则、读写流程、检索策略与边界约束。
  - 实现 `memory_ops.py` 的核心命令（初始化、写入、读取、校验）。
  - 明确与 Git/SDD 的职责边界，避免功能重叠。
- 范围外：
  - 不做模型训练、强化学习、向量数据库接入。
  - 不存储完整 diff、完整会话日志、完整源码快照。
  - 不新增第二套计划状态机（计划状态仍由 SDD 管理）。
- 验收标准：
  - 新 skill 目录结构完整：`SKILL.md`、`agents/`、`references/`、`scripts/`。
  - 能通过命令生成分层记忆并检索 `resume/debug/release` 三类上下文包。
  - 每条关键记忆都包含 Git 锚点和可追溯证据指针。
  - 设计文档与脚本行为一致，且可复现、可审计。
  - 提供触发/不触发场景矩阵，确保不会泛触发。
  - 提供 `retrieve` 输出契约（字段、排序、截断、证据引用规则），保证下游可消费。
  - 提供失败模式与兜底策略（误召回、漏召回、记忆膨胀、语义漂移）。

## 2. 功能拆解
- 模块 A：记忆模型与存储规范
  - 设计 L1/L2/L3 字段与数据约束。
  - 设计 `docs/memory/` 存储结构与索引格式。
- 模块 B：事件采集与关键节点提级
  - 支持关键事件写入、状态快照、节点提级。
  - 通过规则阈值控制“什么进入关键记忆”。
- 模块 C：按需检索与上下文打包
  - 支持基于任务场景的检索策略（恢复、调试、发布）。
  - 输出面向 agent 的紧凑上下文包。
- 模块 D：一致性治理
  - 校验 schema、引用完整性、Git/SDD 锚点一致性。
  - 输出可执行修复建议。
- 模块 E：触发与价值治理
  - 定义触发/不触发矩阵与边界案例。
  - 定义失败模式处理与降级策略。

## 3. 实现方案
- 数据流：
  - 触发信号（里程碑变化、重复失败、关键决策）进入采集流程。
  - `capture` 写入事件，并按规则落到 L1/L2/L3。
  - `retrieve` 按 profile 检索并组装上下文包，返回给 coding agent。
- 命令/契约变更：
  - 新增 CLI：`python scripts/memory_ops.py <subcommand> --root <repo> ...`。
  - 子命令初版包含：`init`、`capture`、`promote`、`snapshot`、`retrieve`、`doctor`。
- 兼容与迁移：
  - v1 为增量能力，不影响现有 `docs/plans`。
  - 与 SDD 通过 `plan_id`/`milestone_id` 关联，不接管状态所有权。
  - 检索失败时降级返回 L1 + 最近高优先级 L2，避免空上下文。

## 4. 技术决策
- 关键设计：
  - 规则优先：先用确定性规则做关键节点识别，再考虑智能优化。
  - 结构化存储：JSON/JSONL，方便 diff、审查、脚本处理。
  - 严格锚点：每条记录包含 `branch`、`head_sha`、证据链接、`plan_id`。
  - 检索契约固定：返回字段、排序权重、条目上限、截断规则固定化。
- 取舍：
  - 规则法可解释性强，但灵活性低于学习法；优先稳定落地。
  - 结构化写入增加少量操作成本，但显著降低恢复与交接成本。
  - 固定契约降低自由度，但显著提升跨 agent 复用稳定性。
- 风险与缓解：
  - 风险：记忆膨胀。缓解：提级阈值、归档策略、后续 `gc`。
  - 风险：语义漂移。缓解：要求证据指针，`doctor` 校验关键字段。
  - 风险：与 SDD/Git 重叠。缓解：在 SKILL 中写明职责边界并强校验。
  - 风险：误触发导致噪音写入。缓解：触发矩阵 + 不触发回归样例。
  - 风险：误召回/漏召回影响可用性。缓解：profile 阈值 + 检索降级策略。

## 6. 测试与验收
- 单元测试：
  - 记录 schema 校验、字段必填校验、命令参数校验。
  - 检索排序和过滤逻辑校验。
- 集成测试：
  - 端到端：`init -> capture -> promote -> snapshot -> retrieve -> doctor`。
  - 与 SDD 关联：`plan_id` 引用、状态不冲突校验。
  - 触发矩阵：触发/不触发场景回归验证。
  - 检索契约：字段完整性、排序稳定性、截断一致性验证。
- 人工验证：
  - 恢复场景：中断后能快速拿到“当前状态 + 关键尝试 + 下一步”。
  - 调试场景：同题多次失败时能召回过往失败原因与证据。
  - 发布场景：只返回高信号风险和决策信息，不引入噪音。
  - 失败兜底场景：检索信号不足时仍能返回可执行最小上下文。
