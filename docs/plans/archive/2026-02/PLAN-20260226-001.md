## 1. Requirement Analysis
- User intent: Build a layered project memory skill that complements SDD and preserves key R&D state for resume, handoff, and focused context retrieval.
- Scope in:
  - Define skill contract and activation gate for layered memory usage.
  - Design three-layer memory model (project state, key events, node insights).
  - Implement deterministic CLI for memory read/write and retrieval profiles.
  - Integrate Git anchors and SDD plan references without ownership overlap.
- Scope out:
  - No model fine-tuning, reinforcement learning, or vector database in v1.
  - No storage of raw full diffs or full interaction transcripts.
  - No second lifecycle state machine parallel to SDD.
- Acceptance criteria:
  - Skill package exists at `skills/layered-project-memory/` with `SKILL.md`, `agents/`, `references/`, and `scripts/`.
  - `memory_ops.py` supports `init`, `capture`, `promote`, `snapshot`, `retrieve`, and `doctor`.
  - Retrieval profiles (`resume`, `debug`, `release`) return compact context packs with evidence pointers.
  - Data schema and boundary rules explicitly enforce "Git facts + SDD lifecycle + memory semantics" separation.

## 2. Functional Decomposition
- Module A: Memory Data Model and Storage Layout
  - Define L1/L2/L3 schemas and index format.
  - Define on-disk structure under `docs/memory/`.
- Module B: Capture and Promotion Workflow
  - Record events with Git/SDD anchors.
  - Promote high-value events by rule thresholds.
- Module C: Retrieval and Context Packing
  - Query relevant events by profile and target scope.
  - Assemble minimal context packs for coding agents.
- Module D: Consistency and Governance Checks
  - Validate schema, references, and anchor integrity.
  - Report drift and missing evidence.

## 3. Implementation Approach
- Data flow:
  - Input signals (user milestone updates, failed attempts, key decisions) feed capture pipeline.
  - Capture writes L1 snapshots + L2 index entries + optional L3 insight files.
  - Retrieve uses profile-specific filters to return ranked and compressed context pack.
- API/contract changes:
  - Add CLI contract: `python scripts/memory_ops.py <subcommand> --root <repo> ...`.
  - Add reusable record schema in references for deterministic field-level validation.
- Migration or compatibility notes:
  - v1 is additive and can run in repos with only SDD assets.
  - Existing `docs/plans/` structure remains source of truth for lifecycle status.

## 4. Technical Solution
- Key design choices:
  - Rule-first event classification (deterministic and auditable).
  - JSON/JSONL storage for portability and diff friendliness.
  - Strict anchor fields: `plan_id`, `milestone_id` (optional), `branch`, `head_sha`, evidence refs.
- Trade-offs:
  - Rule-first approach is less adaptive than learning-based ranking but easier to trust and maintain.
  - Structured memory adds write overhead but reduces recovery and duplicate-debug cost.
- Risks and mitigations:
  - Risk: memory bloat. Mitigation: promotion thresholds + retention policy + `gc` follow-up.
  - Risk: semantic drift in summaries. Mitigation: enforce evidence pointers and doctor checks.
  - Risk: boundary overlap with SDD. Mitigation: explicit owner contract in SKILL.md and schemas.
