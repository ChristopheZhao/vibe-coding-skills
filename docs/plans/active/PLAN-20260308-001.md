## 1. Requirement Analysis
- User intent:
  - 新增一个讨论期专用 skill，用于多智能体协作策略指导。
  - 该 skill 只处理需求讨论/难题讨论，不进入执行规划与运行时编排。
  - 设计需要同时具备抽象原则与 examples，且 examples 不能写死为固定 case。
- Scope in:
  - 创建 `multi-agent-discussion-advisor` skill 目录与核心文档。
  - 定义触发/反触发边界、输出契约、角色选择方法、模式库、平台适配提示。
  - 增加 smoke 检查，验证结构与关键契约。
- Scope out:
  - 不实现运行时 orchestration 引擎。
  - 不实现任务执行分解或里程碑管理。
  - 不替代 memory/experience skill 的存储职责。
- Acceptance criteria:
  - skill 结构满足仓库规范并通过 structure check。
  - skill smoke 通过并包含正反触发边界校验点。
  - 文档清晰体现 discussion-only 边界与“原则+example”双轨。

## 2. Functional Decomposition
- Module A: SKILL 主契约
  - 明确 purpose、activation cues、scope limit、ownership boundary、workflow/hard rules。
- Module B: 原则与信号体系
  - `principles.md` + `trigger-signals.md` + `process-protocol.md`。
- Module C: 模式与角色体系
  - `collaboration-patterns.md` + `role-selection.md` + `output-contract.md`。
- Module D: 示例与回归
  - 参数化 examples + non-examples + regression-cases。
- Module E: 验证与集成
  - `scripts/smoke.sh` 与仓库 `README` 清单更新。

## 3. Implementation Approach
- Data flow:
  - 用户讨论请求 -> 信号判定 -> 选择协作模式与最小角色集 -> 输出讨论建议卡 -> 执行期交接。
- API/contract changes:

  - 新增文件：`SKILL.md`、`references/*`、`references/examples/*`、`scripts/smoke.sh`、`agents/openai.yaml`
  - 新增计划：`docs/plans/PLAN-20260308-001.md`
- Migration or compatibility notes:

  - 与 `sdd-plan-maintainer`、`layered-project-memory`、`experience-capture` 正交协作。

## 4. Technical Solution
- Key design choices:
  - `discussion-only`：限制在讨论期，不侵入执行编排。
  - `principles + examples` 双轨：原则约束上界，example 提供可迁移范式。
  - `signal-driven`：按信号判定触发，不按固定 case 硬编码。
  - `minimal-role-set`：角色最小充分，避免角色膨胀。
- Trade-offs:
  - 先保证边界清晰与触发精度，暂不实现复杂自动化机制。
  - 文档驱动优先，执行层面留给其他 skill。
- Risks and mitigations:
  - 风险：误触发到执行阶段。  
    缓解：在 `SKILL.md` 与 `regression-cases.md` 增加反触发与交接规则。
  - 风险：examples 变成固定路由。  
    缓解：examples 使用参数化模板并声明“非穷举”。
  - 风险：与现有 skill 边界重叠。  
    缓解：ownership boundary 明确职责归属。
